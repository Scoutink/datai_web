How It Works

From upload to audit, fully automated.

CMR DMS / DocAI is built around a repeating loop: organize, control, automate, monitor. Here is how it fits together.

Configure your foundation

Set up branding, category hierarchy, storage backend (local or AWS S3 with encryption), allowed file types, and SMTP delivery configuration.

Build your team and permissions

Create users, define roles, assign role membership by drag-and-drop, apply page-level overrides, and maintain external clients in one directory.

Upload and organize

Upload single or bulk files, apply categories, meta tags, lifecycle state, access visibility, assignments, and expiration dates.

Collaborate, share, and finalize

Share with bounded access, use protected links, add comments, send emails, apply signatures/watermarks, collect external files, and run reminders.

Automate workflows and approvals

Design multi-step transitions, assign owners, track graph progress, and manage org-wide vs personal workflow queues with logs.

Monitor and comply

Search document and login audit logs, inspect email and cron histories, and enforce retention policies across 30–365 day windows.

Before CMR DMS vs After CMR DMS

Before

  • Documents scattered across emails and shared drives
  • Manual email approval chains
  • No version control — overwritten files
  • No record of who accessed or modified what
  • External file collection via email attachments
  • Compliance reporting done manually

After

  • Centralized, searchable library with category hierarchy
  • Automated multi-step workflows with visual tracking
  • Complete version history with one-click restore
  • Full audit trail, searchable by document, user, and date
  • Controlled file request links with type and size limits
  • Automated retention policies and exportable audit logs
